# Security Policy

This repository publishes the public SysAdminDoc GitHub profile README, generated project catalog, and portfolio feed.

## Reporting

Use [GitHub private vulnerability reporting](https://github.com/SysAdminDoc/SysAdminDoc/security/advisories/new) to report security issues privately. Do not include secrets, private repository names, personal data, medical data, customer data, or exploit details in a public issue.

If the report is not security-sensitive, use the public issue forms for broken links, profile corrections, or workflow problems.

## Scope

In scope:

- Public profile README generation and validation.
- Public `projects.json` feed data.
- Public setup snippets, release links, and workflow automation in this repository.

Not handled in this repository:

- Private repositories or private production systems.
- Vulnerabilities in third-party services linked from the README.
- Requests to disclose non-public project names, customer details, medical data, or employer-specific information.

## Response

Security reports are reviewed within 7 business days. Public exposure risks are prioritized and triaged immediately. Confirmed public-profile issues are fixed through normal repository commits without exposing sensitive details.
