# Security Policy

This repository publishes the public SysAdminDoc GitHub profile README, generated project catalog, and portfolio feed.

## Reporting a vulnerability

Use [GitHub private vulnerability reporting](https://github.com/SysAdminDoc/SysAdminDoc/security/advisories/new) to report security issues. Do not include secrets, private repository names, personal data, medical data, customer data, or exploit details in a public issue.

For non-sensitive reports (broken links, profile corrections, validation failures), use the [public issue forms](https://github.com/SysAdminDoc/SysAdminDoc/issues/new/choose) instead.

## Scope

**In scope:**

- Public profile README generation and validation logic.
- Public `projects.json` feed data and schema contracts.
- Public setup snippets, release links, and install one-liners.
- Workflow automation and validation scripts in this repository.

**Not in scope:**

- Private repositories or private production systems.
- Vulnerabilities in third-party services linked from the README.
- Requests to disclose non-public project names, customer details, medical data, or employer-specific information.

## Response timeline

Security reports are reviewed within 7 business days. Public exposure risks (leaked private repo names, credentials, or sensitive data in tracked files) are prioritized and triaged immediately. Confirmed issues are fixed through normal repository commits without exposing sensitive details in the fix description.
